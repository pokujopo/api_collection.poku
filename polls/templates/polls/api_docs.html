Advanced Media Downloader API Documentation

Hati rasmi ya ushirikiano wa mifumo ya kupakua video na audio (Asynchronous Background Task Queue)

Hati hii inatoa maelekezo ya kina ya jinsi ya kuingiliana na API yetu ya kisasa ya kupakua maudhui kutoka mitandao ya kijamii kama YouTube na Facebook. API hii inafanya kazi chini ya mfumo wa sekunde sifuri (Asynchronous) kupitia Celery na Redis.

🔒 Uthibitisho (Authentication)

Maombi yote ya API lazima yaambatane na ufunguo wa ulinzi wa X-API-Key uliowekwa kwenye HTTP Header ya kila request. Kushindwa kuweka ufunguo huu kutasababisha server kutema kosa la 401 Unauthorized.

X-API-Key: e7CG7bz3.0INIHuWSxrbhO4LSvQUMvM2dQIjA0b34
Content-Type: application/json

📥 1. Omba Kupakua Maudhui (Request Media Download)

POST /api/v1/download/

Endpoint hii inapokea link ya video, inasajili kazi chini kwa chini kwenye mfumo wa Celery Worker, na inakurudishia majibu ya papo hapo bila kukusubirisha download imalizike.

Vigezo Vinavyotakiwa (JSON Payload Parameters)

Kigezo (Field) Aina (Type) Lazima? Maelezo (Description)
link String Ndiyo URL halisi ya video kutoka YouTube au Facebook.
format_type String Ndiyo Aina ya faili unalotaka: video au audio.
quality String Ndiyo Ubora wa picha: 1080p, 720p, 360p au best kwa audio.

Mfano wa Bando la Request (JSON Input)

{
    "link": "https://www.facebook.com/share/v/1E62QanCsZ/",
    "format_type": "video",
    "quality": "720p"
}

Mfano wa Amri ya cURL (cURL Snippet)

curl -X POST http://127.0.0.1:8000/api/v1/download/      -H "X-API-Key: YOUR_API_KEY_HERE"      -H "Content-Type: application/json"      -d '{
           "link": "https://www.facebook.com/share/v/1E62QanCsZ/",
           "format_type": "video",
           "quality": "720p"
         }'

Majibu ya Kazi Ikisajiliwa vizuri (201 Created Response)

{
    "id": 52,
    "link": "https://www.facebook.com/share/v/1E62QanCsZ/",
    "format_type": "video",
    "quality": "720p",
    "status": "Pending",
    "name": null,
    "error_message": null
}

📊 2. Kagua Hali ya Download (Check Download Status)

GET /api/v1/download/status/<int:id>/

Kwa sababu kazi inafanyika asynchronous kule nyuma, mfumo wako wa Frontend unatakiwa uwe unapiga kwenye endpoint hii (Polling) kila baada ya sekunde t = 3 kukagua kama faili limekamilika.

Mfano wa Amri ya cURL ya Kukagua Status

curl -X GET http://127.0.0.1:8000/api/v1/download/status/52/      -H "X-API-Key: YOUR_API_KEY_HERE"

Majibu ya Server Kama Kazi Imefanikiwa (Success Response)

{
    "id": 52,
    "status": "Success",
    "name": "Vinicius_Jr_Celebration_52.mp4",
    "download_url": "http://127.0.0.1:8000/media/downloads/Vinicius_Jr_Celebration_52.mp4",
    "error_message": null
}

Majibu ya Server Kama Kazi Imefeli (Failed Response)

{
    "id": 52,
    "status": "Failed",
    "name": null,
    "download_url": null,
    "error_message": "Unsupported URL: Video hii ni ya faragha (Private) au imefutwa."
}

⚠️ ILANI YA USALAMA WA STORAGE: Faili linalopatikana kwenye download_url litadumu kwenye storage kwa muda wa dakika 1 (sekunde 60) pekee kabla ya kufutwa kiotomatiki na Celery Beat Task ili kulinda diski ya server yako isijae.
