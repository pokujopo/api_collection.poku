Poku Media Downloader API Documentation

Hati rasmi ya ushirikiano wa mifumo (Asynchronous Tasks na Supabase Auth System)

Hati hii inatoa maelekezo rasmi ya ki-uhandisi jinsi ya kuunganisha Frontend yetu na API ya backend iliyopo Render. Mfumo huu unatumia teknolojia ya Celery na Redis kuchakata downloads nzito kwa sekunde sifuri, huku data zote zikiwa salama kule Supabase.

🌐 BASE URL YA MFUMO: Njia zote za API zilizotajwa hapa chini zinaanza na mzizi huu:
https://api-collection-poku.onrender.com

🔐 1. Mfumo wa Uthibitisho na Ulinzi (Authentication Chain)

Kabla ya kupiga request yoyote ya download, mtumiaji ni lazima ajisajili, aingie kwenye mfumo kupata Token, na kisha azalishe X-API-Key yake maalum.

Hatua ya A: Kusajili Mtumiaji Mpya (User Registration)

POST /api/v1/register/

Inasajili mtumiaji mpya kule kwenye database ya Supabase na kurudisha login token papo hapo.

{
    "username": "john_dev",
    "email": "john@poku.site",
    "password": "Mypassword123"
}

Hatua ya B: Kuingia Kwenye Mfumo (User Login)

POST /api/v1/login/

Kama mtumiaji alishajisajili, anapiga hapa kupata Token yake halali ya uthibitisho.

{
    "username": "john_dev",
    "password": "Mypassword123"
}

Hatua ya C: Kuzalisha Ufunguo wa Siri (Generate API Key)

POST /api/v1/generate-key/

Hapa unapitisha Token uliyopewa wakati wa Login kwenye Header ili mfumo ukuzalishie X-API-Key ya kulinda download requests.

Header Inayotakiwa: Authorization: Token <wako_hapa>

Majibu Baada ya Kufanikiwa (201 Created)

{
    "status": "Success",
    "message": "Ufunguo wako wa API Key umezalishwa kwa mafanikio!",
    "api_key": "ZNJy8Xtx.S3d7WGBwEE5ZN7IbZs37gObkpyikSWL3"
}

📥 2. Omba Kupakua Maudhui (Request Media Download)

POST /api/v1/download/

Endpoint hii inapokea link, inasajili kazi ki-asynchronous kule nyuma kwa Celery Worker, na inasave data ikiwa Pending kule Supabase.

Ulinzi: Lazima uweke ile API Key uliyopewa kwenye Header kama X-API-Key.

Vigezo Vinavyotakiwa (JSON Payload)

Kigezo (Field) Aina (Type) Lazima? Maelezo (Description)
link String Ndiyo URL halisi ya video kutoka YouTube au mitandao inayokubalika.
format_type String Ndiyo Chagua aina ya maudhui: video au audio.
quality String Ndiyo Ubora: 1080p, 720p, 360p (Acha wazi au weka yoyote kwa audio).

Mfano wa Amri ya cURL

curl -X POST https://api-collection-poku.onrender.com/api/v1/download/ \
     -H "X-API-Key: ZNJy8Xtx.S3d7WGBwEE5ZN7IbZs37gObkpyikSWL3" \
     -H "Content-Type: application/json" \
     -d '{
           "link": "https://www.youtube.com/watch?v=dQw4w9WgXcQ",
           "format_type": "video",
           "quality": "720p"
         }'

📊 3. Kagua Hali ya Download (Check Download Status)

GET /api/v1/download/status/<int:id>/

Frontend inatakiwa iwe inapiga hapa (Polling) kila baada ya sekunde 3 kukagua maendeleo ya download kwa kutumia ID ya kazi.

Mfano wa cURL ya Status

curl -X GET https://api-collection-poku.onrender.com/api/v1/download/status/2/

Majibu Kama Download Imekamilika (Success Response)

{
    "id": 2,
    "link": "https://www.youtube.com/watch?v=dQw4w9WgXcQ",
    "format_type": "video",
    "quality": "720p",
    "status": "Success",
    "name": "download_2.mp4",
    "error_message": null,
    "download_link": "https://api-collection-poku.onrender.com/media/downloads/download_2.mp4"
}

⚠️ ILANI YA USALAMA WA DISKI: Faili halisi lililopo kwenye download_link litadumu kwenye storage ya Render kwa muda uliopangwa pekee kabla ya Celery Beat kulifuta na kubadilisha status yake kuwa Cleaned ili kuzuia diski isijae.
